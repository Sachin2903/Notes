# System Design
System Design is the process of designing the architecture, components, modules, and data of a system to satisfy specified requirements.

>> Why
Scalability → Can it grow?
Performance → Is it fast?
Reliability → Does it fail?
Maintainability → Can we fix/change it easily?
Cost → Can we afford it?
Time → Can we build it quickly?

🔑 Key Difference
Aspect	       HLD 🏗️	               LLD 🔧
Level	    Big picture	    Detailed implementation
Focus	    Architecture	    Code & logic
Audience    Architects,     senior devs	Developers

Example	Services & APIs	Classes & methods

🧠 Easy Way to Remember
HLD = “What system looks like”
LLD = “How system works internally”
